# 价值选股与RSRS择时策略

## 📋 策略概述

这是一个基于聚宽平台的量化交易策略，主要采用**价值选股**的方法，通过筛选小市值、历史涨停记录、启动点分析等维度来选择股票，并结合多种风控机制进行交易。

**来源**：克隆自聚宽文章 [价值选股与RSRS择时](https://www.joinquant.com/post/15002)  
**作者**：K线放荡不羁

## 🎯 核心策略逻辑

### 1. 选股流程

策略采用多层级筛选机制：

1. **基础筛选**
   - 过滤次新股（上市不足375天）
   - 过滤科创板、北交所股票（代码以4、8开头或68开头）
   - 过滤ST股票及退市标签股票
   - 过滤停牌股票
   - 过滤当日涨停/跌停股票

2. **价值筛选**
   - 按市值从小到大排序
   - 选取前1000只小市值股票作为初始池

3. **历史涨停筛选**
   - 统计过去3年（750个交易日）内的涨停次数
   - 筛选出涨停次数最多的前10%股票

4. **启动点分析**
   - 找到每只股票最近一次涨停
   - 从涨停日开始往前查找，找到首次出现阴线（收盘价 < 开盘价）的K线
   - 将该阴线的最低价作为"启动点"
   - 计算当前价格与启动点的偏移量
   - 优先选择最接近启动点的股票（价格偏移量最小）

5. **行业分散**
   - 从筛选后的股票中选择不同行业的股票
   - 最多选择10个不同行业的股票
   - 最终选择前 `stock_num * 2` 只股票

### 2. 交易规则

#### 调仓频率
- **周频调仓**：每周二 10:30 执行整体调仓
- **持仓数量**：默认持有 3 只股票（可通过 `g.stock_num` 调整）

#### 买入规则
- 等权重分配资金
- 如果持仓数量不足，会从目标列表中买入
- 避免重复买入本周已买入的股票

#### 卖出规则
1. **调仓卖出**：股票不在目标列表且不是昨日涨停股时卖出
2. **涨停监控**：昨日涨停的股票，如果今日打开涨停则卖出
3. **止盈**：收益达到100%时卖出
4. **止损**：支持三种止损策略（见下方详细说明）
5. **换手率异常**：检测到缩量或放量时卖出
6. **放量检测**：如果开启 `HV_control`，当日成交量超过历史最大成交量的90%时卖出

### 3. 止盈止损策略

策略支持三种止损模式（通过 `g.stoploss_strategy` 设置）：

- **策略1**：止损线止损
  - 止盈：收益 ≥ 100%
  - 止损：价格 < 成本价 × 0.91（默认）

- **策略2**：市场趋势止损
  - 监控中证500指数（399101.XSHE）
  - 如果指数平均涨跌幅 ≤ 0.93，清仓所有持仓

- **策略3**：联合策略（默认）
  - 先判断市场趋势，如果大盘惨跌则清仓
  - 否则执行止损线止损

### 4. 换手率检测

策略会实时监控持仓股票的换手率：

- **缩量检测**：20日平均换手率 < 0.3% 时卖出
- **放量检测**：实时换手率 > 10% 且是平均换手率的2倍以上时卖出

### 5. 空仓月份管理

- 支持在特定月份（1月、4月）空仓
- 空仓期间可以持有指定的替代标的（如ETF、债券等）
- 通过 `g.pass_april` 控制是否启用

## ⚙️ 配置参数

### 全局变量说明

```python
# 基础配置
g.stock_num = 3                    # 持仓股票数量
g.init_stock_count = 1000         # 初始股池数量
g.up_price = 20                    # 股票单价上限（未启用）

# 历史数据窗口
g.limit_days_window = 3 * 250     # 历史涨停参考窗口期（3年）

# 止损配置
g.run_stoploss = True             # 是否启用止损
g.stoploss_strategy = 3           # 止损策略（1/2/3）
g.stoploss_limit = 0.91           # 止损线（价格/成本价）
g.stoploss_market = 0.93          # 市场趋势止损参数

# 放量检测配置
g.HV_control = False              # 是否启用放量检测
g.HV_duration = 120               # 放量检测周期（交易日）
g.HV_ratio = 0.9                  # 放量阈值（当前量/历史最大量）

# 空仓月份配置
g.pass_april = True               # 是否在1月、4月空仓
g.no_trading_buy = []             # 空仓期间持有的标的
```

## 📅 交易时间安排

策略在以下时间点执行相应操作：

| 时间 | 函数 | 说明 |
|------|------|------|

| 10:00 | `sell_stocks` | 执行止盈止损 |
| 10:30（周二） | `weekly_adjustment` | 周频调仓 |
| 14:20 | `trade_afternoon` | 检查涨停股、换手率、放量等 |

## 🔧 核心函数说明

### 选股相关

- `get_stock_list(context)`: 主选股函数，执行完整的筛选流程
- `get_history_highlimit(context, stock_list, days, p)`: 筛选历史涨停股票
- `get_start_point(context, stock_list, days)`: 计算启动点并排序
- `get_stock_industry(stock)`: 行业分散选择

### 过滤函数

- `filter_new_stock()`: 过滤次新股
- `filter_kcbj_stock()`: 过滤科创板、北交所股票
- `filter_st_stock()`: 过滤ST股票
- `filter_paused_stock()`: 过滤停牌股票
- `filter_limitup_stock()`: 过滤涨停股票
- `filter_limitdown_stock()`: 过滤跌停股票
- `filter_highprice_stock()`: 过滤高价股票（未启用）

### 交易相关

- `buy_security(context, target_list)`: 买入模块
- `close_position(position)`: 平仓函数
- `open_position(security, value)`: 开仓函数
- `weekly_adjustment(context)`: 周频调仓

### 风控相关

- `sell_stocks(context)`: 止盈止损
- `check_limit_up(context)`: 检查涨停股
- `huanshou(context)`: 换手率检测
- `check_high_volume(context)`: 放量检测

### 辅助函数

- `huanshoulv(context, stock, is_avg)`: 计算换手率（实时或平均）
- `today_is_between(context)`: 判断是否为空仓月份

## ⚠️ 注意事项

1. **数据依赖**：策略依赖聚宽平台的数据接口，需要在聚宽环境中运行
2. **基准指数**：使用中证500指数（399101.XSHE）作为基准和止损参考
3. **涨停监控**：策略会特别关注涨停股票，昨日涨停的股票如果今日打开涨停会立即卖出
4. **行业分散**：通过行业分散降低风险，最多选择10个不同行业
5. **代码问题**：
   - 第325行：`g.reason_to_sell is 'limitup'` 应该是赋值 `=` 而不是比较 `is`
   - 第412行：使用了 `datetime.timedelta` 但未导入 `datetime` 模块

## 🔍 策略特点

### 优点
- ✅ 多维度筛选，逻辑清晰
- ✅ 结合价值选股和技术分析
- ✅ 完善的风控机制（止盈止损、换手率监控等）

### 风险提示
- ⚠️ 小市值股票波动较大
- ⚠️ 历史涨停筛选可能过度拟合
- ⚠️ 启动点策略依赖历史数据质量
- ⚠️ 策略在特定市场环境下可能失效
